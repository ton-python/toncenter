Retries for `429` and server errors are enabled by default with exponential backoff. See [Guide](/guide#retries) for retry policy, rate limiting, and session management.

<Note>
  V2 responses include a `@type` field (e.g. `raw.fullAccountState`, `smc.runResult`) from the underlying tonlib schema. The SDK maps these into typed Pydantic models automatically.
</Note>

## Errors

V2 API may return status codes 400, 401, 403, 404, 405, 409, 422, 429, 500, 504, and 542. See [Guide](/guide#errors) for the full exception hierarchy, descriptions, and handling examples.
