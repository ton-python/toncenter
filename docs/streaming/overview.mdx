---
title: Overview
description: Real-time blockchain events over SSE and WebSocket. Decorator-based handler API — both transports multiplex all subscriptions over a single connection.
---

For the raw wire protocol, see the [Streaming API v2 specification](https://gist.github.com/dungeon-master-666/98db8d73e9cd9a1b7802bc06ded5b155).

| Transport            | Description                                                 |
| -------------------- | ----------------------------------------------------------- |
| `ToncenterSSE`       | One-way stream. Single POST request, server pushes events.  |
| `ToncenterWebSocket` | Bidirectional. Dynamic subscribe/unsubscribe after connect. |

## SDK Usage

<Tabs>
  <Tab title="SSE">
    ```python
    import asyncio

    from toncenter.types import Network
    from toncenter.streaming import Finality, ToncenterSSE, TransactionsNotification
    from toncenter.exceptions import ToncenterConnectionLostError

    sse = ToncenterSSE("YOUR_API_KEY", Network.MAINNET)


    @sse.on_transactions(min_finality=Finality.FINALIZED)
    async def on_tx(n: TransactionsNotification) -> None:
        for tx in n.transactions:
            print(tx.get("hash"))


    async def main() -> None:
        try:
            await sse.start(addresses=["EQDtFpEwcFAEcRe5mLVh2N6C0x-_hJEM7W61_JLnSF74p4q2"])
        except ToncenterConnectionLostError as e:
            print(f"Connection lost after {e.attempts} reconnect attempts")
        finally:
            await sse.stop()


    if __name__ == "__main__":
        asyncio.run(main())
    ```
  </Tab>

<Note>
  An API key is **required** for streaming — unlike REST, unauthenticated connections are rejected. Free plans allow up to **2 concurrent streaming connections**; paid plans provide higher limits.
</Note>

## Errors

Streaming-specific exceptions: `ToncenterConnectionLimitError` (connection limit reached — auto-rotates with multiple keys, fatal with single key), `ToncenterStreamingError` (transport error), and `ToncenterConnectionLostError` (reconnect limit exhausted). See [Connection](/streaming/connection#errors) for details and [Reference](/reference#errors) for the full exception hierarchy.
